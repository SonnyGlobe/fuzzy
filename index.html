A Fuzzy Matching Tool

Ortiz_Fuzzy Lookupis a smart Excel fuzzy matching tool that helps you compare two sheets in your Excel file and find the closest matches automatically. It's especially useful for tasks like data cleaning, merging lists, or cross-referencing datasets.

How It Works

  1. Source Sheet / Column:This is the reference data (like your lookup array in VLOOKUP). Ortiz_Fuzzy will search for matches from this column.
  2. Target Sheet / Column:This is the main sheet you want to compare. Each row in this column will be matched against the source column.
  3. Match Mode Options:
    • Target → Source:Each target row is matched against the source array (default).
    • Source → Target:Each source row is matched against the target array.
    • Both:Ortiz_Fuzzy computes both directions and chooses the best match.
  4. Result:Two new columns are added to your target sheet:
    • BestMatch_[SourceSheet]- The closest match from the source column.
    • MatchScore_[SourceSheet]- The match percentage score.

Step-by-Step Usage

  1. Upload your Excel file (.xlsx) containing both the Source and Target sheets.
  2. Enter theSource Sheet NameandSource Column(e.g., A).
  3. Enter theTarget Sheet NameandTarget Column(e.g., I).
  4. Select theMatch Mode:
    • Target → Source(default)
    • Source → Target
    • Both
  5. Set amatch threshold(default is 80%). Only matches with a score equal to or higher than the threshold will be recorded.
  6. Click theRun Fuzzy Lookupbutton.
  7. Download the processed Excel file. The last two columns of the target sheet will contain thebest matchandmatch scorefor each row.

Tips & Notes

  • Empty or invalid cells are ignored automatically.
  • Ortiz_Fuzzy preserves all other sheets in your Excel file.
  • Use column letters (A, B, C…) or numbers (1, 2, 3…) to specify columns.
  • You can adjust the match threshold depending on how strict you want the matching to be.
  • Large Excel files may take longer to process; a higher threshold reduces false matches.

Note: Ortiz_Fuzzy Lookup is best suited for textual data, such as names, addresses, or product codes. It may not be accurate for numeric-only comparisons.
